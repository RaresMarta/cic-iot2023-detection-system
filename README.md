# RT-IDS for CIC-IoT-2023

Bachelor thesis project: a real-time intrusion detection system for IoT networks, trained and evaluated on the CIC-IoT-2023 dataset (39-feature release) and exposed through a Gradio demo.

## What's in here

| Path | Purpose |
| --- | --- |
| `ids_pipeline.ipynb` | Main pipeline — ingestion, deduplication, sampling, three split strategies (temporal, per-capture, random), three label granularities (2/8/34-class), MLP training, tree baselines, latency benchmark. |
| `demo/` | Gradio app wrapping the trained artefacts. CSV and PCAP upload paths. See `demo/README.md` for runtime setup. |
| `data/CSV/CSV/` | Raw CIC-IoT-2023 CSVs (one folder per attack class). Not committed. |
| `data/cic_iot_2023.parquet` | Cached deduplicated/subsampled corpus. Regenerated by deleting it. |
| `models/` | Saved scaler, label encoders, and model state dicts (one triple per (split, granularity)). |
| `docs/report/main.pdf` | Thesis report. Built from `docs/report/main.tex` with `pdflatex` + `bibtex`. |

## Pipeline

Open `ids_pipeline.ipynb` and run top-to-bottom. The configuration block near the top exposes the per-class row cap, the active splits and granularities, the training hyperparameters, and the deterministic seed. An iteration mode (one split, one granularity) and a full thesis-run mode (three splits, three granularities) are documented inline.

The pipeline writes its artefacts into `models/` under a uniform filename pattern that encodes the split and granularity; the demo discovers them at startup.

## Demo

```
python -m demo.app
```

Binds at `http://localhost:7860`. The variant dropdown lists every artefact triple discovered in `models/`. Two upload tabs: pre-extracted CIC-format CSV, or raw `.pcap`/`.pcapng` (extracted via CICFlowMeter, configured per `demo/README.md`).
